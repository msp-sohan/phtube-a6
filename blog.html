1. Discuss the scope of var, let, and const

1. JavaScript var keyword: The Var keyword is a global variable. Variables declared with var can be re-declared and reassigned.

Scope: The scope of the var keyword is the global or function scope. It means variables defined outside the function can be accessed globally, and variables defined inside a particular function can be accessed within the function.

Example: Variable ‘a’ is declared globally. So, the scope of the variable ‘a’ is global, and it can be accessible everywhere in the program. The output shown is in the console.

Javascript

var a = 10
var a = 100
console.log(a)

var x = 10
x = 1000
console.log(x);

Output:

100
1000

2. JavaScript let keyword: Just like var, variables declared with let can be reassigned to other values, but they cannot be re-declared.

Scope: This is Block Scoped: The scope of a let variable is only block scoped. It can’t be accessible outside the particular block ({block}). Let’s see the below example.

Example: Users cannot re-declare the variable defined with the let keyword but can update it.

Javascript

let a = 10
let a = 10
console.log(a)

let x = 10
x = 10
console.log(x)

Output:

Uncaught SyntaxError: Identifier 'a' has already been declared
10

3. JavaScript Const keyword: const is different from var and let. const is used for declaring constant variables – which are variables with values that cannot be changed.

Scope: Constants are block-scoped, much like variables defined using the let statement. The value of a constant cannot change through re-assignment, and it can't be re-declared.

Example: Variable ‘a’ is declared globally. So, the scope of the variable ‘a’ is global, and it can be accessible everywhere in the program. The output shown is in the console.

Javascript

const number1 = 50
const number1 = 100
console.log(number1)

const number2 = 50
number2 = 100
console.log(number2);

Output:

// SyntaxError: Identifier 'number' has already been declared
// TypeError: Assignment to constant variable

2. Tell us the use cases of null and undefined

Null

What is Null?

Null literally means no value. The same meaning is analogous to the meaning of null in programming. Null is basically an assignment value to a variable. A variable assigned as null has no value.

When we assign null as a value to any variable, it means that it is empty or blank. It is to show that the variable has no value. Also, null is an object in JavaScript. When it gets assigned to a variable, it represents no value. The setting of the value must be done manually by the user as JavaScript never sets the value as null. An object can be emptied by setting it to null.

Syntax of Null in JavaScript

var x = null;

Here we have assigned the value null to variable x.

Undefined

What is Undefined?

There is a subtle difference between null and undefined, but as a programmer, it is important that we understand it clearly. As the name suggests, undefined means "not defined". So we declare a variable but do not assign a value to it, the variable becomes undefined.

Unlike null, the value of an undefined variable is set by JavaScript as undefined. The variable gets created at the run-time. When we do not pass an argument for a function parameter, the default value is taken as undefined. Besides, when a function doesn't return a value, it returns undefined.

Syntax of Undefined in JavaScript

var x;

Here we don't assigned the value to variable x.

3. What do you mean by REST API?

REST API?

REST API stands for Representational State Transfer Application Programming Interface. It is an architectural style for an API that uses HTTP requests to access and use data. REST APIs are widely used in web development and other distributed systems.

The full meaning of REST is as follows:
  • Representational: The data in a REST API is represented in a resource-oriented way. This means that each piece of data is represented as a resource, and each resource has its own unique URI.

  • Stateless: REST APIs are stateless, which means that the server does not store any information about the client's state. This makes REST APIs more scalable and efficient.

  • Transfer: The data in a REST API is transferred using the HTTP protocol. HTTP is a well-known and well-supported protocol, which makes REST APIs easy to use and secure.

  • API-Application Programming Interface: A REST API is an application programming interface (API). This means that it is a set of rules that define how software applications can interact with each other.
